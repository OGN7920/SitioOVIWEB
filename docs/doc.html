GIT

En este aparte de esta unidad se revisaran los contenidos basicos de GIT y sus conceptos.

que es GIT, lo que se tiene que entender, es que GIT es un sistema distribuido no hay un repositorio central, todas la copias que exiten son un branch del proyecto. Cada usuario tiene una copia del proyecto que puede manipulara para poder revisar y realizar los cambios que esten a su alcance lo que esto genera es una mejora del trabajo total, por lo tanto es una exelente herramienta para el trabajo en equipo.

Cuando hablamos de los cambios se definen como COMMIT, cuando no es centralizado, se ha un merge, entre ellos sin afectar a mas a nadie, esto es central para para el workflow usando GIT.

REPOSITORIO: En este aparte revisaremos los las copias de los avances realizados.

BRANCH : Es una linea seperada y tiene su propia historia

ETIQUETAS: apunta a linea de tiempo par realizar o descartar cambios

COMMIT son los cambios que se realizan en un repositorio

REVISION ; identifica la revision de un cambio ejecutado.
